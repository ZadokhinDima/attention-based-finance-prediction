# Attention-Based Finance Prediction

Diploma thesis research comparing attention-based deep learning architectures for multi-horizon S&P 500 log-return forecasting.

## Overview

This project benchmarks five model families on a multi-variate financial time series task:

| Model | Architecture | Complexity |
|-------|-------------|------------|
| LSTM | Recurrent | O(L) |
| Transformer | Vanilla self-attention | O(L²) |
| Informer | ProbSparse attention | O(L log L) |
| FEDformer | Frequency-domain attention (Fourier / Wavelet) | O(L) |
| iTransformer | Inverted attention (variables as tokens) | O(N²) |

Each model is evaluated in two configurations:
- **Full**: 29 S&P 500 tickers × 5 OHLCV features = 145 input features
- **Simple**: S&P 500 index only, 5 OHLCV features

## Task

**Multi-horizon log-return regression** — predict log-returns at t+1, t+3, and t+10 simultaneously.

- **Universe**: 29 large-cap S&P 500 constituents (see `tickers.csv`)
- **Period**: 2010–2025
- **Lookback window**: 60 trading days
- **Evaluation**: Walk-forward cross-validation (11 folds, 2015–2025, expanding window)
- **Metrics**: MAE, MSE, Directional Accuracy per horizon + aggregate

## Methodology

1. **Hyperparameter search** — grid search on a fixed train/val split (2010–2014 train, 2014–2015 val)
2. **Walk-forward validation** — 11 expanding folds from 2015 to 2025, each fold adds ~1 year of training data
3. **Best config** — lowest validation MAE from HP search, applied to all 11 folds

## Requirements

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install torch numpy pandas scikit-learn matplotlib yfinance einops sympy pywavelets
```

The notebook uses [FEDformer](https://github.com/MAZiqing/FEDformer) and [iTransformer](https://github.com/thuml/iTransformer) for their frequency-domain layers. Clone them into `repos/`:

```bash
mkdir repos
git clone https://github.com/MAZiqing/FEDformer repos/FEDformer
git clone https://github.com/thuml/iTransformer repos/iTransformer
```

## Results

Full walk-forward results are generated by running the notebook end-to-end. The final cells produce a summary table and per-horizon comparison plots across all models.
